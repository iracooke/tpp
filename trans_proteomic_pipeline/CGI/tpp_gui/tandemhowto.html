Processing X!Tandem Results

IPP has some file name and file location requirements when processing X!Tandem results. When X!Tandem is run from the IPP interface the user does not need to be aware of these requirements. If you run X!Tandem from the IPP interface, IPP will do all the file conversions automatically and place the files where IPP can find them.

If you are importing X!Tandem result into IPP you should be aware of the following requirements:

  • All the data files for a given experiment should be in the same directory (mzXML, pepXML, X!TandemXML and .dta folders).
  • All MS data and analysis results files should have the same base name.
  • The results in X!Tandem format should end with 'tandem.xml'.
  • The results in pepXML format should end with '.xml'.
  • Folders containing the .dta should be in the data directory (see the Tools/Convert mzXML Files tabs for .dta conversion).
  • The parameters file used to run X!Tandem must contain the following lines.

    <note type="input" label="output, xsl path"> http://localhost/schema/tandem-style.xsl</note>"

    <note type="input" label="output, path hashing">no</note>

  • The name and location of the .mzXML data file should be defined by a full path in the X!Tandem parameters file.
  • The X!Tandem results file should be defined by a file name without a path in the X!Tandem parameters file.
